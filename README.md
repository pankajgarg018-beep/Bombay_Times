# BombayTimes – Playwright Automation Suite

End-to-end validation framework for [bombaytimes.com](https://www.bombaytimes.com) built with **Python + Playwright + pytest**.

---

## What It Tests

| Area | Validations |
|---|---|
| Category navigation | Entertainment, Lifestyle, Dining, Travel, People, Specials, Astro, BT Picks |
| Article detail pages | Bollywood, Photo Story, BT Picks, Intimate Diaries, Festival, Astro Trends |
| Canonical URLs | Every navigated page – matches opened URL |
| Google Analytics | GA / GTM network calls fire on every page |
| AMP pages | Home Page, Bollywood, Photo Story, BT Picks, Intimate Diaries, Festival, Astro Trends |
| Author pages | Author link clickability, redirect, canonical + GA |
| Sitemap & RSS | 8 sitemap URLs + 1 RSS feed – HTTP 200 + valid XML |
| Search flow | Search bar opens, keyword submitted, results page reached |

**28 automated test cases** run sequentially and produce a full HTML + PDF report.

## Project Structure

```
BT_New/
├── conftest.py              # Pytest fixtures, browser setup, HTML/PDF report generation, email sender
├── pytest.ini               # Pytest settings (logging, test path, markers)
├── requirements.txt         # Pinned Python dependencies
├── send_report.py           # Standalone script: generate PDF + email the report manually
├── diagnose_detail_pages.py # Developer utility: inspect DOM selectors (not part of test run)
│
├── locators/
│   └── home_locators.py     # CSS selector constants for homepage elements
│
├── pages/
│   ├── base_page.py         # BasePage: shared navigation helpers (goto, click, click_role …)
│   └── home_page.py         # HomePage: all category/menu click methods
│
├── utils/
│   └── helpers.py           # safe_click() utility (available for use in custom extensions)
│
├── tests/
│   └── test_bombaytimes_flow.py  # All 28 test cases
│
└── reports/
    ├── bt_report.html        # Generated after each run
    ├── bt_report.pdf         # PDF version (generated by send_report.py)
    └── test.log              # Full debug log
```

## Setup

```bash
# 1. Create and activate a virtual environment
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS / Linux

# 2. Install dependencies
pip install -r requirements.txt

# 3. Install Playwright browser (uses system Chrome if available)
playwright install chromium
```

---

## Running Tests

```bash
# Run the full suite (28 tests, ~10 min)
python -m pytest

# Run with minimal output
python -m pytest --tb=no -q

# Run a single test
python -m pytest tests/test_bombaytimes_flow.py::test_bollywood_article_detail_flow

# Run two tests together (e.g. detail + AMP)
python -m pytest tests/test_bombaytimes_flow.py::test_bollywood_article_detail_flow tests/test_bombaytimes_flow.py::test_amp_page_validation
```

The HTML report is saved to `reports/bt_report.html` after every run.

---

## Sending the Report by Email

```bash
# Set your Gmail App Password, then run:
$env:EMAIL_PASSWORD = "your-16-char-app-password"   # PowerShell
python send_report.py
```

The script generates a PDF from the HTML report and emails it to the configured recipient.  
See `_EMAIL_CONFIG` in `conftest.py` or `send_report.py` for SMTP settings.

---

## Email Configuration

Edit `_EMAIL_CONFIG` in `conftest.py` (for automatic post-run emails) or the constants at the top of `send_report.py` (for manual sending):

```python
_EMAIL_CONFIG = {
    "smtp_host": "smtp.gmail.com",   # or smtp.office365.com for Office 365
    "smtp_port": 587,
    "use_tls":   True,
    "from_addr": "your-sender@gmail.com",
    "username":  "your-sender@gmail.com",
    "password":  "",   # set via EMAIL_PASSWORD env var
    "to_addrs":  ["recipient@example.com"],
}
```

## Notes

- Tests run in **headed** Chrome (browser is visible) — change `headless=False` → `headless=True` in the `browser` fixture inside `conftest.py` for CI/headless mode.
- Selectors prefer ARIA roles with href fallbacks; update them if the site markup changes.
- Test order is fixed (file order); `@pytest.mark.order(1)` pins the first test.
- Screenshots of failures are embedded in the HTML report.
